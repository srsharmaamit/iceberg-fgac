🏭Production Stack Differences

In production: MinIO is replaced by NetApp StorageGRID (S3-compatible). PostgreSQL is a managed RDS or on-prem HA cluster. Spark runs on a dedicated cluster (Kubernetes or YARN). Airflow runs on a managed platform. Nessie connects to the same PostgreSQL cluster but in its own schema. Everything else stays the same.

2Create MinIO Bucket Structure

Iceberg writes all Parquet data files, manifests, and metadata JSON to MinIO. The bucket must exist before Nessie or Spark can create any table.

2.1 — Understand the warehouse layout

s3a://warehouse/ ├── nessie/ # Nessie catalog warehouse root │ ├── bronze/raw_transactions/ │ │ ├── data/ # Parquet files (created by Iceberg) │ │ └── metadata/ # JSON + manifests (created by Iceberg) │ ├── silver/ │ │ ├── cleansed_transactions/ │ │ └── cleansed_customers/ │ ├── gold/ │ │ ├── account_summary/ │ │ └── jurisdiction_risk_report/ │ ├── entitlements/rules/ # FGAC policy table │ └── audit/query_log/ # Audit log table └── hive/ # HMS secondary catalog (optional)
ℹ️

You only need to create the top-level warehouse bucket. Iceberg creates data/ and metadata/ subfolders automatically on first write.

2.2 — Create bucket via MinIO Console (UI)

  1. Open http://localhost:9001 → login minioadmin / minioadmin
  2. Click Buckets → Create Bucket
  3. Name: warehouse — leave versioning OFF
  4. Click Create Bucket
  5. Repeat with name hive-warehouse if using HMS

2.3 — Or via MinIO Client CLI

docker exec -it minio sh -c "
  mc alias set local http://localhost:9000 minioadmin minioadmin
  mc mb local/warehouse
  mc mb local/hive-warehouse
  mc ls local
"

2.4 — Verify Spark can read/write S3A

🏭Production — NetApp StorageGRID

Replace http://minio:9000 with your NetApp S3 gateway endpoint (e.g. https://s3.netapp.internal). Create a dedicated IAM service account with a scoped access key — do not use admin credentials. Grant the service account s3:GetObject, s3:PutObject, s3:DeleteObject, s3:ListBucket on the warehouse bucket only. NetApp ACLs then enforce object-level isolation; FGAC enforces query-level isolation on top.

3Set Up Nessie Namespaces and Branches

Nessie is your versioned data catalog. Branches work exactly like Git — main is production, domain branches are staging lanes, and per-run branches are isolated write zones.

3.1 — Branch strategy diagram

main # production — never written to directly ├── transactions/main # transactions staging │ └── transactions/dev # transactions active dev │ └── etl-txn-20260316 # per-run branch (Airflow creates this) └── customers/main └── customers/dev └── etl-cust-20260316

3.2 — Create branches via Nessie REST API

NESSIE=http://localhost:19120/api/v1

# Get main branch HEAD hash
MAIN_HASH=$(curl -s $NESSIE/trees/tree/main \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['hash'])")
echo "main HEAD: $MAIN_HASH"

# Create all domain branches
for BRANCH in transactions/main transactions/dev customers/main customers/dev; do
  curl -s -X POST "$NESSIE/trees/tree" \
    -H "Content-Type: application/json" \
    -d "{\"name\": \"$BRANCH\", \"type\": \"BRANCH\", \"hash\": \"$MAIN_HASH\"}" \
    | python3 -c "import sys,json; d=json.load(sys.stdin); print(f'  Created: {d[\"name\"]}')"
done

3.3 — Verify all branches

curl -s http://localhost:19120/api/v1/trees \
  | python3 -c "
import sys, json
for t in json.load(sys.stdin)['references']:
    print(f'  {t[\"type\"]:8}  {t[\"name\"]:30}  {t[\"hash\"][:12]}')
"

3.4 — Create namespaces via Spark SQL save as setup_namespaces.py

  .config("spark.sql.catalog.nessie.uri",           "http://nessie:19120/api/v1")
  .config("spark.sql.catalog.nessie.ref",           "main")
  .config("spark.sql.catalog.nessie.warehouse",      "s3a://warehouse/")
  .config("spark.sql.catalog.nessie.authentication.type", "NONE")
  .config("spark.hadoop.fs.s3a.endpoint",            "http://minio:9000")
  .config("spark.hadoop.fs.s3a.access.key",          "minioadmin")
  .config("spark.hadoop.fs.s3a.secret.key",          "minioadmin")
  .config("spark.hadoop.fs.s3a.path.style.access",   "true")
  .config("spark.hadoop.fs.s3a.impl", "org.apache.hadoop.fs.s3a.S3AFileSystem")
  .getOrCreate())

for ns in ["bronze", "silver", "gold", "entitlements", "audit"]:
    spark.sql(f"CREATE NAMESPACE IF NOT EXISTS nessie.{ns}")
    print(f"  ✓ nessie.{ns}")

spark.sql("SHOW NAMESPACES IN nessie").show()

# Submit the setup script
docker exec -it spark-master spark-submit \
  --master spark://spark-master:7077 \
  /opt/spark/jobs/setup_namespaces.py
🏭Production — Branch Protection

Document and enforce: ETL jobs never write directly to main. Every Airflow DAG must create a per-run branch, write to it, validate, and merge. Periodically promote domain/dev → domain/main → main after QA sign-off. In a future Nessie version, role-based branch protection will be enforceable via the API.

✅ Step 3 complete when:
  • Nessie /api/v1/trees shows main, transactions/dev, customers/dev (and main variants)
  • SHOW NAMESPACES IN nessie returns bronze, silver, gold, entitlements, audit

4Place the FGAC Fat JAR

The fat JAR contains the FGAC framework plus all its dependencies (Spring, Guava, Iceberg Java API). It must be on the Spark driver classpath before any job referencing the FGAC extension can run.

4.1 — Copy JAR into the Spark container

# Copy from your host machine into the Spark master container
docker cp fgac-spark-adapter-1.0.0-SNAPSHOT-spark-fat.jar \
  spark-master:/opt/spark/jars/fgac-spark-adapter.jar

# Verify it landed correctly
docker exec -it spark-master ls -lh /opt/spark/jars/fgac-spark-adapter.jar
💡

Placing the JAR in /opt/spark/jars/ makes it available to ALL jobs on this master automatically. No need to pass --jars on every spark-submit call.

4.2 — Also copy the seed policies file

# Copy your seed-policies.json into the container so SeedPolicyLoader can find it
docker cp config/policies/seed-policies.json \
  spark-master:/opt/spark/conf/seed-policies.json

# Verify
docker exec -it spark-master cat /opt/spark/conf/seed-policies.json | head -20

4.3 — Verify the JAR is valid

# Check the FGAC extension class exists inside the JAR
docker exec -it spark-master jar -tf /opt/spark/jars/fgac-spark-adapter.jar \
  | grep FgacSparkExtension
# Expected: com/banking/lakehouse/fgac/spark/extension/FgacSparkExtension.class
🏭Production — JAR Distribution

In production, do not docker cp manually. Instead: publish the fat JAR to a Maven repository (Nexus/Artifactory) or object storage. In your Spark cluster startup script, pull the JAR from that location. Use versioned JAR names (fgac-spark-adapter-1.0.0.jar) so rollback is a one-line config change. All Spark worker nodes need the JAR too if using cluster mode — add it to your cluster provisioning scripts.

✅ Step 4 complete when:
  • ls -lh /opt/spark/jars/fgac-spark-adapter.jar shows the file (~80-100 MB)
  • jar -tf shows FgacSparkExtension.class inside
  • seed-policies.json is in /opt/spark/conf/

5Define Your Entitlement Policies

Policies live in seed-policies.json. This file defines who can access which table with what row filter and which columns masked. On first Spark startup, SeedPolicyLoader reads this file and creates the nessie.entitlements.rules Iceberg table.

5.1 — Policy file structure

// /opt/spark/conf/seed-policies.json
{
  "version": "1.0.0",
  "description": "Banking lakehouse entitlement rules",
  "entitlements": [

    {
      // Data engineers — full unrestricted access for ETL
      "roleName":       "data_engineer",
      "tableName":      "silver.cleansed_transactions",
      "rowPredicate":   "ALLOW_ALL",
      "allowedColumns": [],
      "maskedColumns":  {},
      "redactedColumns":[],
      "auditEnabled":   true
    },

    {
      // Compliance officers — all rows but PII masked
      "roleName":       "compliance_officer",
      "tableName":      "silver.cleansed_transactions",
      "rowPredicate":   "ALLOW_ALL",
      "allowedColumns": [
        "transaction_id", "account_number", "transaction_amount",
        "transaction_ts", "jurisdiction", "merchant_category"
      ],
      "maskedColumns": {
        "account_number": "fgac_mask_account_number"
      },
      "redactedColumns": [],
      "auditEnabled": true
    },

    {
      // UK analyst — UK rows only, account number masked
      "roleName":      "analyst_uk",
      "tableName":     "silver.cleansed_transactions",
      "rowPredicate":  "jurisdiction = 'UK'",
      "allowedColumns": [
        "transaction_id", "account_number", "transaction_amount",
        "transaction_ts", "jurisdiction", "merchant_category"
      ],
      "maskedColumns": { "account_number": "fgac_mask_account_number" },
      "redactedColumns": ["data_classification", "_ingested_at"],
      "auditEnabled": true
    },

    {
      // EU analyst — EU jurisdictions only
      "roleName":      "analyst_eu",
      "tableName":     "silver.cleansed_transactions",
      "rowPredicate":  "jurisdiction IN ('DE','FR','NL','ES','IT','BE','PL')",
      "allowedColumns": [
        "transaction_id", "account_number", "transaction_amount",
        "transaction_ts", "jurisdiction"
      ],
      "maskedColumns": { "account_number": "fgac_mask_account_number" },
      "redactedColumns": ["data_classification"],
      "auditEnabled": true
    }
  ]
}

Store seed-policies.json in your company's Git repository under data-platform/policies/. Require a pull request with data governance team approval for any change. Use CI to validate JSON schema. 6Bootstrap the Policy Store

The first time any Spark job runs with the FGAC extension loaded, SeedPolicyLoader automatically creates nessie.entitlements.rules and seeds it. But it's cleaner to run a dedicated bootstrap job so you know exactly when and how this happened.

6.1 — Create the bootstrap script

# bootstrap_policy_store.py
from pyspark.sql import SparkSession

spark = (SparkSession.builder.appName("fgac-bootstrap")
  # FGAC extension
  .config("spark.sql.extensions",
    "com.banking.lakehouse.fgac.spark.extension.FgacSparkExtension,"
    "org.apache.iceberg.spark.extensions.IcebergSparkSessionExtensions,"
    "org.projectnessie.spark.extensions.NessieSparkSessionExtensions")
  # Service account identity
  .config("spark.app.current_user", "svc_airflow")
  # Seed file location
  .config("spark.driver.extraJavaOptions",
    "-Dfgac.nessie.uri=http://nessie:19120/api/v1 "
    "-Dfgac.nessie.warehouse=s3a://warehouse/ "
    "-Dfgac.s3a.endpoint=http://minio:9000 "
    "-Dfgac.s3a.access.key=minioadmin "
    "-Dfgac.s3a.secret.key=minioadmin "
    "-Dfgac.policy.seed.path=file:///opt/spark/conf/seed-policies.json")
  # Nessie catalog
  .config("spark.sql.catalog.nessie",              "org.apache.iceberg.spark.SparkCatalog")
  .config("spark.sql.catalog.nessie.catalog-impl",  "org.apache.iceberg.nessie.NessieCatalog")
  .config("spark.sql.catalog.nessie.uri",           "http://nessie:19120/api/v1")
  .config("spark.sql.catalog.nessie.ref",           "main")
  .config("spark.sql.catalog.nessie.warehouse",      "s3a://warehouse/")
  .config("spark.sql.catalog.nessie.authentication.type", "NONE")
  # S3A
  .config("spark.hadoop.fs.s3a.endpoint",            "http://minio:9000")
  .config("spark.hadoop.fs.s3a.access.key",          "minioadmin")
  .config("spark.hadoop.fs.s3a.secret.key",          "minioadmin")
  .config("spark.hadoop.fs.s3a.path.style.access",   "true")
  .config("spark.hadoop.fs.s3a.impl", "org.apache.hadoop.fs.s3a.S3AFileSystem")
  .getOrCreate())

print("Verifying policy store was seeded...")
# The FGAC extension seeded the table on startup — verify by querying it
result = spark.sql("SELECT role_name, table_name, row_predicate FROM nessie.entitlements.rules ORDER BY role_name")
result.show(50, truncate=False)
print(f"Total entitlement rules: {result.count()}")
spark.stop()
# Submit the bootstrap job
docker exec -it spark-master spark-submit \
  --master spark://spark-master:7077 \
  --jars /opt/spark/jars/fgac-spark-adapter.jar \
  /opt/spark/jobs/bootstrap_policy_store.py

9Write the Banking ETL DAG in Airflow

The ETL pipeline follows the medallion architecture: Bronze (raw ingest) → Silver (cleansed) → Gold (aggregated). Each run uses a Nessie branch so data is isolated until validation passes.

9.1 — DAG structure overview

                    ┌─────────────────────┐
                    │  create_run_branch   │  creates etl-txn-YYYYMMDD off transactions/dev
                    └──────────┬──────────┘
                               │
                    ┌──────────▼──────────┐
                    │  ingest_to_bronze    │  raw CSV/JSON → nessie.bronze.raw_transactions
                    └──────────┬──────────┘
                               │
                    ┌──────────▼──────────┐
                    │  cleanse_to_silver   │  bronze → nessie.silver.cleansed_transactions
                    └──────────┬──────────┘
                               │
                    ┌──────────▼──────────┐
                    │  aggregate_to_gold   │  silver → nessie.gold.account_summary
                    └──────────┬──────────┘
                               │
                    ┌──────────▼──────────┐
                    │  validate_row_counts │  assert counts match expectations
                    └──────────┬──────────┘
                               │
                ┌──────────────▼──────────────────┐
                │  merge_branch_to_dev             │  merge etl-txn-YYYYMMDD → transactions/dev
                └──────────────┬──────────────────┘
                               │
                    ┌──────────▼──────────┐
                    │  drop_run_branch     │  cleanup
                    └─────────────────────┘

9.2 — Full DAG file

# dags/banking_etl_dag.py
from airflow import DAG
from airflow.providers.apache.spark.operators.spark_submit import SparkSubmitOperator
from airflow.operators.python import PythonOperator
from airflow.utils.dates import days_ago
from datetime import datetime, timedelta
import requests

DAG_ID       = "banking_etl_pipeline"
NESSIE_URL   = "http://nessie:19120/api/v1"
SPARK_CONN   = "spark_default"
SPARK_JAR    = "/opt/spark/jars/fgac-spark-adapter.jar"

default_args = {
    "owner":          "data-platform",
    "retries":         1,
    "retry_delay":     timedelta(minutes=5),
    "email_on_failure": True,
    "email":           ["data-alerts@bank.internal"],
}

def get_run_branch(**ctx):
    return f"etl-txn-{ctx['ds_nodash']}"  # e.g. etl-txn-20260316

def create_nessie_branch(**ctx):
    branch = get_run_branch(**ctx)
    base   = "transactions/dev"
    base_hash = requests.get(f"{NESSIE_URL}/trees/tree/{base.replace('/','%2F')}").json()["hash"]
    resp = requests.post(f"{NESSIE_URL}/trees/tree",
        json={"name": branch, "type": "BRANCH", "hash": base_hash})
    resp.raise_for_status()
    print(f"Created branch: {branch}")

def merge_nessie_branch(**ctx):
    branch = get_run_branch(**ctx)
    hash_ = requests.get(f"{NESSIE_URL}/trees/tree/{branch.replace('/','%2F')}").json()["hash"]
    resp = requests.post(f"{NESSIE_URL}/trees/tree/transactions%2Fdev/merge",
        json={"fromRefName": branch, "fromHash": hash_})
    resp.raise_for_status()
    print(f"Merged {branch} → transactions/dev")

def drop_nessie_branch(**ctx):
    branch = get_run_branch(**ctx)
    hash_ = requests.get(f"{NESSIE_URL}/trees/tree/{branch.replace('/','%2F')}").json()["hash"]
    requests.delete(f"{NESSIE_URL}/trees/branch/{branch.replace('/','%2F')}?expectedHash={hash_}")
    print(f"Dropped branch: {branch}")

# Common Spark config applied to every task
def spark_conf(ds_nodash):
    return {
        "spark.app.current_user": "svc_airflow",
        "spark.sql.catalog.nessie.ref": f"etl-txn-{ds_nodash}",  # write to run branch!
        "spark.driver.extraJavaOptions":
            "-Dfgac.nessie.uri=http://nessie:19120/api/v1 "
            "-Dfgac.nessie.warehouse=s3a://warehouse/ "
            "-Dfgac.s3a.endpoint=http://minio:9000 "
            "-Dfgac.s3a.access.key=minioadmin "
            "-Dfgac.s3a.secret.key=minioadmin"
    }

with DAG(DAG_ID, default_args=default_args,
         schedule_interval="0 2 * * *",  # 2am daily
         start_date=days_ago(1), catchup=False) as dag:

    create_branch = PythonOperator(
        task_id="create_run_branch", python_callable=create_nessie_branch)

    bronze = SparkSubmitOperator(
        task_id="ingest_to_bronze", conn_id=SPARK_CONN,
        application="/opt/airflow/spark_jobs/ingest_bronze.py",
        jars=SPARK_JAR,
        conf=spark_conf("{{ ds_nodash }}"))

    silver = SparkSubmitOperator(
        task_id="cleanse_to_silver", conn_id=SPARK_CONN,
        application="/opt/airflow/spark_jobs/cleanse_silver.py",
        jars=SPARK_JAR,
        conf=spark_conf("{{ ds_nodash }}"))

    gold = SparkSubmitOperator(
        task_id="aggregate_to_gold", conn_id=SPARK_CONN,
        application="/opt/airflow/spark_jobs/aggregate_gold.py",
        jars=SPARK_JAR,
        conf=spark_conf("{{ ds_nodash }}"))

    merge_branch = PythonOperator(
        task_id="merge_branch_to_dev", python_callable=merge_nessie_branch)

    drop_branch = PythonOperator(
        task_id="drop_run_branch", python_callable=drop_nessie_branch)

    create_branch >> bronze >> silver >> gold >> merge_branch >> drop_branch
🏭Production DAG Hardening

13.4 — What to monitor in production

  • DENIED audit events — high volume indicates a misconfigured role or a probing attack
  • Nessie container memory — set JVM heap: -Xmx2g for production. Watch for GC pressure.
  • MinIO / NetApp disk usage — Iceberg never deletes old data files automatically without expire_snapshots
  • Audit table partition count — should grow by ~1 partition/day. Large gaps mean audit flush is broken.
  • FGAC policy cache miss rate — visible in Spark driver logs. High miss rate degrades query latency.
  • Nessie PostgreSQL storage — Nessie metadata grows linearly with commits. Monitor DB size.

🏭Production — Runbook Essentials

If FGAC extension fails to load: Spark will refuse to start all sessions — check driver logs for Spring context errors, ensure the fat JAR is on the classpath and Nessie is reachable. If audit log stops receiving events: check IcebergAuditSink.isHealthy() — if false, Nessie is unreachable; events fall back to LoggingAuditSink automatically. If policy cache never refreshes: check that IcebergPolicyStore.reload() is not throwing — usually means a Nessie connectivity blip. Never restart Spark to fix a policy — use cache invalidation instead.
