1 · Overview

Banks and financial institutions running Apache Iceberg lakehouses need row-level and column-level access control that is enforced transparently — without requiring analysts to write security predicates themselves, and without trusting applications to apply them correctly.

iceberg-fgac is a Java framework that plugs into the Spark Catalyst query optimizer as a custom optimizer rule. Before any data is physically read, the framework rewrites every Iceberg table scan in the logical plan to inject the entitlements for the authenticated user:

🔒

Row Filtering

SQL predicates injected as Iceberg Expression objects — pushed into the scan planner before files are listed.

🎭

Column Masking

Sensitive columns wrapped with UDF calls. Analysts see ****1234 instead of raw account numbers.

📋

Immutable Audit

Every query — allowed or denied — written to an Iceberg table with 7-year retention.

⚡

Sub-microsecond

Policy cache in memory (Guava LoadingCache). Zero I/O on the hot path.

🔌

Engine-agnostic API

Core interfaces have zero Spark dependency. A Trino adapter needs only implement 3 methods.

📦

Versioned Policies

Entitlement rules stored in an Iceberg table in Nessie — time-travelable and branch-testable.

ℹ️
PoC Scope This is a Proof of Concept for an on-premise banking lakehouse. Service accounts (svc_airflow, svc_spark) are the only identity type. No Kerberos or OIDC is wired in yet — the identity resolver reads from spark.app.current_user in the Spark session config.

2 · Architecture

The diagram below shows the full system stack. FGAC is enforced on the driver side during Spark's Catalyst optimization phase — before any tasks are dispatched to executors.

Full Stack Architecture

Request Flow — What happens when an analyst runs a query

1
Airflow submits Spark job

SparkSubmitOperator sets spark.app.current_user=svc_airflow in the job config. This is the identity token read by SparkIdentityResolver.

2
Spark session starts — extension loads

FgacSparkExtension.apply() is called. Spring context is initialised on the driver. CatalystEntitlementRule is registered via injectOptimizerRule.

3
Analyst query hits Catalyst

CatalystEntitlementRule.apply(plan) is called. It walks the logical plan bottom-up via transformUp looking for DataSourceV2Relation Iceberg scan nodes.

4
Policy lookup — sub-microsecond

PolicyCacheManager.findMostPermissive(identity, tableName) checks the in-memory Guava cache. On a miss it reads from nessie.entitlements.rules via the Iceberg Java API and caches for 5 minutes.

5
Plan rewriting

If a row predicate exists: new Filter(parsedExpr, relation) is injected. Iceberg's Spark connector then pushes this into its scan planner as a native Expression — only matching files are listed. If columns are restricted: new Project(allowedCols, plan) is injected with masked columns wrapped in UDF calls.

6
Audit emitted asynchronously

AsyncAuditEventEmitter.emit(event) places an event on a bounded queue and returns immediately (zero latency impact). A background thread drains the queue every 10 seconds and appends batches to nessie.audit.query_log.

3 · Module Structure

9 Maven modules arranged in a strict dependency order. No circular dependencies. fgac-api is the only module engine adapters need to compile against.

Module Dependency Graph
fgac-api

Pure Java interfaces and immutable domain objects. Zero framework dependencies. The only module a future Trino adapter needs.

fgac-core

IcebergExpressionBuilder (predicate parser), PolicyCacheManager (Guava TTL cache), ColumnPolicyEngine, Spring context lifecycle.

fgac-policy-store

Reads/writes entitlement rules from nessie.entitlements.rules via the Iceberg Java API. Seeds on first startup from JSON.

fgac-masking

6 pure static masking functions — no engine dependency. Account number, email, national ID, phone, DOB, sort code.

fgac-audit

Bounded BlockingQueue emitter, batch-write to nessie.audit.query_log. Never blocks the query path.

fgac-spark-adapter

Catalyst rule, FgacSparkExtension, UDF registration, identity resolver. Produces a fat JAR with Shade plugin.

fgac-testing

MockPolicyStore, CapturingAuditEmitter, PolicyFixtures — shared test infrastructure, no external services needed.

fgac-bom

Bill of Materials — version alignment for consumers importing individual modules rather than the full stack.

fgac-integration-tests

Testcontainers-based end-to-end tests (skeleton). Requires Docker. Run separately from unit tests.

4 · Design Decisions

Every key decision made during this project, with the rationale locked in.

Access Control Engine

  • Java framework with engine-agnostic core Rejected: Apache Ranger (too heavyweight, requires HDFS), OPA (JSON policy language, separate process, network hop per query), Lake Formation (AWS-only). The Iceberg Expressions API gives us pushdown-aware row filtering with zero additional dependencies.
  • Catalyst optimizer rule — not a DataSource wrapper Injecting at the optimizer level means the filter appears in the logical plan before physical planning. Iceberg's Spark connector sees it as a native expression and pushes it to the file-level manifest filter. A DataSource wrapper would apply after file listing — much less efficient.

Policy Storage

  • Policies stored as an Iceberg table (nessie.entitlements.rules) Versioned, time-travelable, branchable — same tooling as the data. A policy change on a Nessie branch can be tested before merge. 7-year retention via Iceberg snapshot expiry aligns with banking audit requirements.
  • In-memory cache with 5-minute TTL Guava LoadingCache with sub-microsecond reads on hit. Policy changes take effect within 5 minutes without restarting any process.

Nessie Branching Strategy

  • Per-domain branches: transactions/dev → staging → main Each ETL run branches off domain/dev. Changes merge through staging before reaching main, mirroring a software release pipeline applied to data.

Identity

  • Service accounts only — identity via spark.app.current_user PoC scope. For production: replace SparkIdentityResolver with a Kerberos or JWT-backed resolver. The resolver interface is in fgac-api — swap without touching any other module.

Build

  • Iceberg 1.9.2 (not 1.10.x) 1.10.0 introduces format-version V3 (binary deletion vectors, new data types). V3 tables cannot be read by older engines. For a PoC, V3 adds risk with no benefit. 1.9.2 is the latest stable V2-format release for Spark 3.5.
  • Spring Framework (not Spring Boot) for DI Spring Boot autoconfiguration is inappropriate in a Spark fat JAR — it triggers component scanning and embedded server startup. Bare spring-context gives us @Component + @PostConstruct without the boot baggage.
  • Maven Shade plugin with Guava relocation Spark ships its own Guava version. Relocating com.google.guava to com.banking.fgac.shaded.guava in the fat JAR prevents class conflicts at runtime.

5 · FGAC Model

Every entitlement is a triple: a role + a table + a policy. The policy has two parts: a row predicate and a column policy.

Entitlement Object Model

Row Policy Predicate Syntax

The IcebergExpressionBuilder parses a SQL WHERE clause fragment and converts it to an Iceberg Expression. Supported syntax:

SyntaxIceberg APIExample
ALLOW_ALLExpressions.alwaysTrue()Data engineers, compliance officers
DENY_ALLExpressions.alwaysFalse()Default when no entitlement found
col = 'val'Expressions.equal()jurisdiction = 'UK'
col != 'val'Expressions.notEqual()status != 'CANCELLED'
col IN ('a','b')Expressions.in()jurisdiction IN ('DE','FR','NL')
col NOT IN (...)Expressions.notIn()jurisdiction NOT IN ('TEST')
col IS NULLExpressions.isNull()deleted_at IS NULL
col IS NOT NULLExpressions.notNull()effective_from IS NOT NULL
col > nExpressions.greaterThan()risk_score > 7
A AND BExpressions.and()jurisdiction='UK' AND classification='INTERNAL'
A OR BExpressions.or()jurisdiction='UK' OR jurisdiction='IE'

6 · Policy Store

Entitlement rules live in an Iceberg table at nessie.entitlements.rules. The table is created automatically on first startup by SeedPolicyLoader from the JSON seed file.

Schema

ColumnTypeNotes
role_nameSTRING NOT NULLMatches role names in identity system
table_nameSTRING NOT NULLFully qualified: namespace.table
row_predicateSTRING NOT NULLALLOW_ALL, DENY_ALL, or SQL predicate
allowed_columnsSTRING (JSON array)Empty = all columns allowed
masked_columnsSTRING (JSON object){"col": "udf_name"}
redacted_columnsSTRING (JSON array)Completely hidden from output
audit_enabledBOOLEANDefault: true
effective_fromLONG (epoch ms)Default: epoch 0
effective_toLONG (epoch ms)null = no expiry
descriptionSTRINGFree text for operators

Seed File Format

// config/policies/seed-policies.json

  "version": "1.0.0",
  "entitlements": [
    {
      "roleName":      "analyst_uk",
      "tableName":     "silver.cleansed_transactions",

      "maskedColumns":  { "account_number": "fgac_mask_account_number" },

      "auditEnabled":  true

7 · Masking UDFs

All masking functions are pure static methods in MaskingFunctionLibrary — no Spark dependency, testable in isolation. They are registered as Spark SQL UDFs via SparkUdfRegistrar using session.udf().register().

UDF NameInputOutputBanking Use Case
fgac_mask_account_number 1234567890123456 ************3456 UK sort-code accounts, IBANs
fgac_mask_email john.doe@bank.com j*******@bank.com Customer PII in KYC tables
fgac_redact_national_id AB123456C REDACTED NI numbers, passport IDs
fgac_mask_phone +447911123456 +4479111234** Customer contact details
fgac_mask_date_of_birth 1985-04-23 1985-**-** GDPR-sensitive DOB fields
fgac_mask_sort_code 20-00-00 **-**-00 UK sort code + account pairs

UDFs can also be called directly in Spark SQL:

SELECT
  transaction_id,
  fgac_mask_account_number(account_number) AS account_number,
  transaction_amount,
  jurisdiction
FROM nessie.silver.cleansed_transactions
WHERE jurisdiction = 'UK';

8 · Audit Log

Every table scan — whether allowed or denied — produces an audit event. Events are written to nessie.audit.query_log, partitioned by day(event_ts) for efficient time-range queries.

⚖️
7-Year Retention The audit table's Iceberg snapshot expiry is set to 2,555 days. This satisfies FCA/PRA banking data retention requirements in the UK.

9 · Integrating with Spark 3.5

Step 1 — Build the fat JAR

mvn clean package -pl fgac-spark-adapter -am -DskipTests

# Output fat JAR (Shade plugin appends -spark-fat classifier)
fgac-spark-adapter/target/fgac-spark-adapter-1.0.0-SNAPSHOT-spark-fat.jar

Step 2 — Configure the SparkSession

# pyspark / Airflow DAG
spark = (
    SparkSession.builder
    .appName("banking-etl")

    # 1. Register the FGAC extension FIRST in the list
    .config("spark.sql.extensions",
            "com.banking.lakehouse.fgac.spark.extension.FgacSparkExtension,"
            "org.apache.iceberg.spark.extensions.IcebergSparkSessionExtensions,"
            "org.projectnessie.spark.extensions.NessieSparkSessionExtensions")

    # 2. Set the service account identity
    .config("spark.app.current_user", "svc_airflow")

    # 3. FGAC framework config (overrides fgac-defaults.properties)
    .config("spark.driver.extraJavaOptions",
            "-Dfgac.nessie.uri=http://nessie:19120/api/v1 "
            "-Dfgac.s3a.endpoint=http://minio:9000 "
            "-Dfgac.s3a.access.key=minioadmin "
            "-Dfgac.s3a.secret.key=minioadmin")

    # 4. Nessie catalog
    .config("spark.sql.catalog.nessie",         "org.apache.iceberg.spark.SparkCatalog")
    .config("spark.sql.catalog.nessie.catalog-impl", "org.apache.iceberg.nessie.NessieCatalog")
    .config("spark.sql.catalog.nessie.uri",      "http://nessie:19120/api/v1")
    .config("spark.sql.catalog.nessie.ref",      "main")
    .config("spark.sql.catalog.nessie.warehouse", "s3a://warehouse/")
    .config("spark.sql.catalog.nessie.authentication.type", "NONE")
    .getOrCreate()
)

Step 3 — Submit with fat JAR

spark-submit \
  --jars fgac-spark-adapter-1.0.0-SNAPSHOT-spark-fat.jar \
  --conf spark.sql.extensions=com.banking.lakehouse.fgac.spark.extension.FgacSparkExtension,... \
  --conf spark.app.current_user=svc_airflow \
  your_job.py
⚠️
Jar Order Matters The fat JAR must be on the driver classpath before Iceberg and Nessie JARs. If you use spark.jars.packages as well, add the fat JAR explicitly with --jars to ensure it takes precedence.

Java / Scala API (non-Python)

SparkConf conf = new SparkConf()
    .set("spark.sql.extensions",
         "com.banking.lakehouse.fgac.spark.extension.FgacSparkExtension,"
         + "org.apache.iceberg.spark.extensions.IcebergSparkSessionExtensions")
    .set("spark.app.current_user", "svc_airflow");

11 · Hive Metastore (Dual Catalog)

The PoC writes to both Nessie and HMS from the same Spark session. This demonstrates that existing BI tools connecting via HMS can coexist with the Nessie-governed lakehouse.

# SparkSession config for dual-write
.config("spark.sql.catalog.hive_catalog",       "org.apache.iceberg.spark.SparkCatalog")
.config("spark.sql.catalog.hive_catalog.type",  "hive")
.config("spark.sql.catalog.hive_catalog.uri",   "thrift://hive-metastore:9083")
.config("spark.sql.catalog.hive_catalog.warehouse", "s3a://warehouse/hive/")
-- Write the same data to both catalogs
INSERT INTO nessie.silver.cleansed_transactions SELECT * FROM staging;
INSERT INTO hive_catalog.silver.cleansed_transactions SELECT * FROM staging;
ℹ️
FGAC applies to both catalogs CatalystEntitlementRule intercepts any DataSourceV2Relation in the plan regardless of which catalog it originated from, as long as it is an Iceberg table.

14 · Policy Management at Runtime

Adding a new entitlement via Spark SQL

INSERT INTO nessie.entitlements.rules
  (role_name, table_name, row_predicate,
   allowed_columns, masked_columns, redacted_columns,
   audit_enabled, description)
VALUES (
  'analyst_sg',
  'silver.cleansed_transactions',
  'jurisdiction = ''SG''',
  '["transaction_id","account_number","transaction_amount","jurisdiction"]',
  '{"account_number":"fgac_mask_account_number"}',
  '["data_classification","_ingested_at"]',
  true,
  'Singapore analyst — masked account numbers'
);

The change takes effect within fgac.policy.cache.ttl-minutes (default 5 min). For immediate effect without restarting:

// Java / Spark driver code — force cache invalidation
FgacCoreContext.getInstance().getPolicyCacheManager().invalidateAll();

Revoking access

DELETE FROM nessie.entitlements.rules
WHERE role_name = 'analyst_sg'
  AND table_name = 'silver.cleansed_transactions';

Time-travel audit query

-- Who accessed silver.cleansed_transactions in the last 7 days?
SELECT
  principal_id,
  outcome,
  row_filter_applied,
  COUNT(*) AS query_count,
  MAX(event_ts) AS last_access
FROM nessie.audit.query_log
WHERE table_name = 'silver.cleansed_transactions'
  AND event_ts >= UNIX_MILLIS(CURRENT_TIMESTAMP - INTERVAL 7 DAYS)
GROUP BY principal_id, outcome, row_filter_applied
ORDER BY last_access DESC;
